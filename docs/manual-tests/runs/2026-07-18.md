## Run A — iPhone / iOS Safari

**Reference device:** iPhone 14 or newer, iOS 17+, Safari (private window OK).
**Why:** WebKit + iOS gesture nav + safe-area insets are the most likely
source of finicky bugs.

### Cold load & layout
- [x] Open the home URL in a fresh tab. Map tiles render within ~3s, no white flash.
- [x] No content is hidden behind the notch or the home indicator.
- [x] Address bar is visible at first; scrolling the bottom sheet up causes it to collapse without the layout jumping (no `100vh` / `100dvh` pop).
- [x] Rotate to landscape and back to portrait — layout adapts cleanly, no overlap.

### Map & search
- [x] Pinch-zoom and two-finger pan work; the page itself doesn't pan.
- [x] Tap the search input. Virtual keyboard opens; the search input stays visible (not hidden under the keyboard).
- [x] Type a city (e.g. "Lyon"). Suggestions appear within ~500ms of stopping typing.
- [x] Tap a suggestion → map pans / zooms to the result.
- [x] Dismiss keyboard (tap outside or "Done"). Bottom sheet position is unaffected.

### Bottom sheet (the finicky one)
- [x] Tap a church marker → bottom sheet opens at the **middle** snap point.
- [x] Drag the handle up to the **top** snap point. The content area inside the sheet scrolls (not the page).
- [x] At the top snap, scroll the content all the way to the bottom — overscroll bounce is contained inside the sheet, doesn't drag the map.
- [x] At the top snap, scroll back up; the sheet doesn't snap down accidentally.
- [x] Drag the handle down past the middle snap → sheet collapses.
- [ ] Tap a different marker while sheet is open — content updates, snap state stays at middle.
- [x] Close the sheet (drag down to bottom / tap outside / X if any). No marker stays "selected" visually.

### Navigation modal
- [x] Open the menu (burger / nav button). Modal overlays the map.
- [x] Tap outside the modal — closes.
- [x] Tap "Nous contacter" — leaves to confessio.fr/contact. Hit back: returns to the map at the same state (zoom, marker selection).

### Deep links / SEO routes
- [x] Open `/diocese/paris` directly. Page resolves to the map zoomed/bounded on Paris within ~2s. Browser back works (does not loop).
- [x] Open a `/church/<uuid>` URL directly. Page does not crash. (Currently bare; just verify no error screen.)
- [x] Hit a non-existent slug, e.g. `/diocese/zzz-not-real`. App shows a 404, not a server error.

### Edge cases
- [x] Toggle airplane mode briefly while on the map. Reconnect: app recovers (markers reload or state stays).
- [x] Background the app for 1 minute, return: state is preserved, no white screen.

## This release — new features & fixes to verify

Run every item on **both** devices. Mark the `iPhone` and `Android` box on each.

### 1. Bottom sheet persists across navigation
*Changed: map routes moved into a `(map)` route group; the sheet stays mounted across navigations instead of remounting.*
- Open a church so the sheet is at the middle/top snap, then navigate (tap a diocese link / deep-link `/diocese/paris` / browser back-forward). The sheet must **not** flash-remount or reset its snap/scroll state.
- Tap a marker → `/church/<uuid>` deep link → back: the sheet content and map state stay coherent, no double-mount flicker.
  - [x] iPhone
  - [x] Android

### 2. Church card feedback — "Ces informations sont-elles à jour ?"
*Changed: feedback reframed as a question with two labelled pills; submitting now sends `church_uuid`.*
- Open a church card and scroll to the feedback block. It reads **"Ces informations sont-elles à jour ?"** with two pills: **Oui** (👍 + count) and **Erreur** (👎 + count).
- Tap **Erreur** → an error-type chooser + comment box opens; the box scrolls fully above the keyboard (not hidden under it).
- Submit a report → success state shows, box closes, and the vote count reflects it. (No 400/500 — the report includes `church_uuid`.)
- Tap **Oui** → toggles/submits the positive report cleanly.
  - [x] iPhone
  - [x] Android

### 3. Holiday-period schedule warning (scoped to selected day)
*Added: a warning on the church card when the schedule may change during a holiday period; now scoped to the day you've selected.*
- Find a church whose schedule falls in a holiday period and confirm the card shows **"Horaires susceptibles de changer …"** (e.g. "pendant les vacances scolaires").
- Change the selected day via the date rail: the warning appears/disappears according to whether *that day* is in the holiday period — it is not a blanket always-on banner.
  - [x] iPhone
  - [x] Android

### 4. Date filter rail
*Changed: default chip renamed to "Tous les jours"; active day chips toggle off; sheet heading is date-aware.*
- The rail shows **"Tous les jours"** first, then **"Aujourd'hui"** and upcoming day chips.
- Tap a day chip → it becomes active and the sheet heading reflects that day.
- Tap the **same active chip again** → it toggles off and returns to "Tous les jours".
- Horizontal scroll of the rail is smooth; the active chip is legible.
  - [x] iPhone
  - [x] Android

### 5. Search pill focus stability
*Fixed: search pill no longer shifts height/alignment on focus (h-11, full-height input).*
- Tap the search input and type: the pill keeps a constant height, and the text/placeholder doesn't jump vertically on focus/blur.
  - [ ] iPhone
  - [x] Android

### 6. "Nous soutenir" link removed
*Changed: the "Nous soutenir" entry was removed from the navigation modal.*
- Open the navigation modal — **"Nous soutenir" is gone**; "Nous contacter" is still present and works.
  - [x] iPhone
  - [x] Android

## Optional — quarterly extended run

Run roughly once every 3 months, or after any change to layout / sheet / map code:

- **iPad Safari** — verify the desktop layout vs mobile layout breakpoint behaves correctly. Tablet viewports often fall in a "neither" state.
- **Firefox Android** — Gecko is third-most-common on Android. Run the same checklist as Run B; flag anything Firefox-specific.
- **Older iPhone (SE 2nd gen, iOS 16)** — small viewport + older WebKit. Useful for catching layout overflow bugs.

## Reporting issues

For every `❌`:
1. Note OS / browser version exactly (Settings → About).
2. Screen-record if possible (iOS: Control Center; Android: built-in recorder).
3. File a Linear ticket with the recording, the reproduction steps from this checklist, and a link to the run file in `runs/`.
